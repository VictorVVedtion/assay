# assay · 照妖镜

[![ci](https://github.com/VictorVVedtion/assay/actions/workflows/ci.yml/badge.svg)](https://github.com/VictorVVedtion/assay/actions/workflows/ci.yml)

> 给 LLM 中转站照个原形。**独立、开源、自托管**地核验你买到的**用量**和**模型身份**是否名副其实——不信任中转站后台,不用交出 root 账号。
>
> *Independent, self-hosted verification that an LLM API relay (“中转站”) actually delivers the model and the token usage you pay for.*

> ⚠️ **状态:Alpha / 施工中。** Phase 0 实现进行中。总体设计见 [DESIGN.md](DESIGN.md),冻结契约见 [PHASE0.md](PHASE0.md)。

## 它解决什么

中转站(API 转售 / 聚合)是中间人,有两件事它自己说了算、你无从核实:

- **用量真实性** — token 计费由它自报,可虚报输入/输出,可对缓存命中照样收钱。
- **模型身份真实性** — 你买旗舰(GPT-4o / Claude Opus / DeepSeek-R1),它可静默降级、量化、套壳开源模型,照旗舰价收费。

Cloudflare AI Gateway 之类解决不了:要你交账号、闭源、不在数据流上做独立校验——只是把信任从中转站转移给了它。`assay` 反过来:审计器跑在**你自己**机器上,在请求链路上独立取证。

## 工作原理

```
  你的 app ──实时请求──▶ ┌───────────────────────────┐
                        │ 数据面 Go (透传, 毫秒级)     │ ← fail-open: 绝不阻断你的请求
                        │ 转发 → 流式原样回传          │
                        │ → tee 一份完整副本           │
                        └───────────────────────────┘
                  转发 │                  │ 异步副本
                       ▼                  ▼
                   中转站→上游     evidence.jsonl (哈希链, append-only)
                                          │ tail
                                          ▼
                                  分析面 Python ── token_recount / cache_replay / throughput
                                          ▼
                                  verdicts.jsonl (派生, 可复现)
```

- **数据面(Go)**:OpenAI 兼容透传代理,对客户端完全透明(流式 Flush),同时 tee 完整副本。**fail-open**——审计组件再怎么故障,也绝不阻断或拖慢你的请求。
- **分析面(Python)**:tail 证据日志,跑检测,出裁决。在热路径之外,可慢可深。
- **证据日志**:append-only + 哈希链。你信的不是我们的"结论",而是"**任何人都能拿日志里的原始字节重算出同样结论**"(`assay-analyzer replay`)。

## Phase 0 做什么(免基准地板:零可信上游即可用)

| check | 作用 | 适用 |
|---|---|---|
| `token_recount` | tiktoken 本地重算用量,抓虚报 | OpenAI 系(强);其他 skip |
| `provenance` | 被动核验响应头/标记,判断是否真代理到声称的上游(抓套壳苗头) | Claude/OpenAI/Gemini |
| `exposure` | 度量你**实际送出去**的敏感内容下界(密钥/PII/代码),请求+响应双向扫 | 全部 |
| `cache_replay` | 抓"同一响应喂多个不同请求"的缓存重放计费 | 全部 |
| `throughput` | 速度/吞吐采集 + 物理不可能粗筛 | 全部(遥测) |

加上哈希链证据 + `assay verify` / `assay-analyzer replay` 的可复现核验,以及**对自身证据日志的凭据脱敏**(body 里的 key 在落盘前替换成 `[assay-redacted:类型]`,真 key 仍照常发给上游)。

> 模型身份的深度检测(LLMmap 主动指纹、MMD 分布检验、社区指纹库)在 Phase 1+。路线见 [DESIGN.md §7](DESIGN.md)。

## 关于"中转站会不会卖我数据"(读这段,别被任何人骗)

这是真实且有实测数据的威胁:一项对 28 个付费 + 400 个免费中转站(正是 new-api/sub2api 那套)的测量发现,有的注入恶意代码、有的盗用 prompt 里的 AWS 凭据、有的偷加密钱包私钥;中文财经媒体也报道过运营者被兜售"卖用户数据"。([来源见 DESIGN.md](DESIGN.md))

但你必须先认清一个**物理事实**:

> **架在链路上的代理,阻止不了中转站读你的数据。** 中转站必须解密明文才能转发给上游——这是中间人(MITM)的固有属性。**任何声称"我能防止中转站读/卖你数据"的产品,都是在骗你。** 唯一的真·防护是**根本不把这份数据经过它**(直连厂商,或用它读不了的端到端加密——但那样它也没法转发了)。

所以 `assay` 在数据保密上**只做能诚实兑现的三件事**,且每件都标清边界:

1. **度量你在泄露什么**(`exposure` check)— 扫描你实际送出的密钥/PII/代码,给一个**下界**计数。这是**测量,不是防护**;"检测到 0"**不等于**"安全"(检测器会漏新格式、模糊人名、可推断身份)。它能帮你**主动少送**,仅此而已。
2. **不让 assay 自己成为第二个泄漏源** — 证据日志里 body 的凭据在落盘前脱敏;头部密钥脱敏;支持只含哈希/裁决、不含明文的"可分享脱敏导出"。注意:加密静存只防离线备份泄漏,**防不了**运行中的代理(它本来就持有明文)或同机私钥。
3. **(Phase 1+)canary 诱饵事后检测** — 往合成探针里埋唯一诱饵,被触碰则**实锤证明**数据被留存/滥用。但:**只能证明命中,无法证明安全**(静默归档转售可能永不触发);命中也**无法证明是谁**(中转站/上游/下游买家无法区分)。

**做不到的(终局,需厂商配合,assay 给不了):** TEE 机密计算推理(连厂商都读不到明文)是唯一真·隐藏明文的技术,但它是**推理端点**的属性,而中转站按定义不是端点——买家侧代理处在信任边界的错误一侧,无法提供也无法验证它。对国内用户额外提醒:数据经中转站送往境外厂商,本身可能构成 **PIPL 数据出境**事件。

## 快速开始

> 🚧 实现完成后补全可运行步骤。预期形态:

```bash
cp assay.example.yaml assay.yaml      # 填上你的中转站地址
docker compose up -d                  # 起 proxy + analyzer
export OPENAI_BASE_URL=http://localhost:8080/v1   # 你的 app 指过来即可
```

## 诚实的边界

- **归因鸿沟**:检测输出是"与正品分布不一致 + 证据",**不是**"诈骗"定论——差异也可能来自良性量化/微调/推理非确定性。
- **用量核验只对 OpenAI 强**:Claude/Gemini 无公开分词器,只能信转发数字或往返厂商估算——这是硬限制,非实现缺口。
- **没有密码学铁证**:单请求级铁证需要厂商对响应签名,不在本项目范围(终局,需厂商配合)。

完整限制见 [DESIGN.md §8](DESIGN.md)。

## 隐私

证据日志含**明文 prompt 与输出**(你最敏感的数据)。默认**仅本地**,Authorization 等头部脱敏。这正是 `assay` 必须自托管、**不能**做成 SaaS 的根本原因。
